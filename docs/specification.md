> 最終更新: 2026-03-09

## 1. プロジェクト概要

**Dr.stretch SPOT** は、Dr.ストレッチを退職した認定トレーナー（OB/OG）向けのスポットバイトマッチングプラットフォームです。退職後も技術を活かせる副業機会を提供し、店舗の人手不足をリアルタイムで解消します。

### 解決する課題

| 課題 | 対象 | SPOT の解決策 |
|------|------|--------------|
| 急な欠勤・人手不足 | 店舗 | 即日シフト募集 → 自動マッチング |
| 退職後の副業機会がない | トレーナー | スキルを活かしたスポットバイト |
| 人件費の見える化 | HR・経営 | リアルタイムコスト分析 + 予算上限管理 |

### 技術スタック

| レイヤー | 技術 |
|---------|------|
| フレームワーク | Next.js 16 (App Router, Turbopack) |
| UI | shadcn/ui + Tailwind CSS v4 |
| データベース | Supabase (PostgreSQL + Auth + RLS) |
| ホスティング | Vercel (Serverless) |
| LINE Bot | LINE Messaging API + @line/bot-sdk |
| メール | Resend API |
| 言語 | TypeScript (strict) |

## 2. ユーザーロールと権限

| ロール | 説明 | ログイン方法 |
|--------|------|-------------|
| トレーナー | 退職トレーナー。シフト応募・打刻・収入管理 | OTP（メール認証コード） |
| 店舗マネージャー | 店舗責任者。シフト作成・応募管理・評価入力 | デモクイックログイン |
| エリアマネージャー | 複数店舗管轄。HR権限の一部 | デモクイックログイン |
| HR（人事部） | 全体管理。時給・ルール設定・直接オファー | デモクイックログイン |
| Admin | システム管理者。全権限 | デモクイックログイン |

### アクセス制御（RLS）

全テーブルで Row Level Security を有効化。ロールに応じたポリシーで、トレーナーは自分のデータのみ、店舗は自店舗のみ、HR/Admin は全データにアクセス可能。

## 3. 主要機能

### 3-1. シフトマッチング（コア機能）

```
① 店舗マネージャーがシフト作成
   → status: pending_approval

② HR が承認
   → status: open（トレーナーに公開）

③ トレーナーが応募
   → 時給自動計算（基本時給 + 出勤ボーナス + 緊急ボーナス）
   → auto_confirm=true の店舗: 自動承認 → シフト確定
   → auto_confirm=false: 店舗マネージャーがレビュー

④ シフト確定
   → 出退勤レコード自動作成
   → メール通知 + LINE通知（連携済みの場合）

⑤ 全員分埋まったら
   → shift_requests.status → closed（自動）
```

### 3-2. HR 直接オファー

HR が特定のトレーナーに直接シフトオファーを送信できる機能。

```
① HR がトレーナーを検索（エリア・ランク・ブランク状態でフィルタ）
② オファー作成（日付・時間・店舗・時給自動計算）
③ トレーナーに通知
   → Web アプリ内通知
   → LINE Push通知（Flex Message + 承諾/辞退ボタン）
④ トレーナーが承諾/辞退
   → 承諾: シフト確定（shift_request + application + attendance 自動作成）
   → 辞退: オファー辞退記録
⑤ HR ダッシュボードに結果反映
```

### 3-3. QR 打刻（出退勤管理）

```
出勤:
  ① トレーナーがアプリで「QRコード生成」
  ② 店舗スタッフがQR読み取り → 出勤記録
  ③ GPS位置情報で店舗範囲内か自動検証

退勤:
  ① 店舗が「退勤リクエスト」
  ② トレーナー側にQR自動表示（5秒ポーリング）
  ③ 店舗が読み取り → 退勤記録
```

### 3-4. 時給計算ロジック

```
確定時給 = 基本時給（在籍年数ベース）
         + 出勤ボーナス（過去30日のスライディング実績）
         + 緊急ボーナス（24h以内 & 50%未満の埋まり率）
```

- 時給は「応募時点で固定」され、後からの設定変更に影響しない
- コスト上限（max_hourly_rate）による自動キャップあり
- 時給設定は `hourly_rate_config` テーブルで管理、HR が変更可能

### 3-5. ブランク管理（無勤務日数）

| ステータス | 条件 | 制限 |
|-----------|------|------|
| ok | 60日未満 | なし |
| alert_60 | 60日以上 | 通知のみ |
| skill_check_required | 90日以上 | 応募ブロック（スキルチェック必須） |
| training_required | 120日以上 | 応募ブロック（研修必須） |

- pg_cron で毎日自動判定（02:00 JST）
- 日数閾値は `blank_rule_config` テーブルで変更可能

### 3-6. トレーナーランクシステム

| ランク | 条件 |
|--------|------|
| Bronze | 初期ランク |
| Silver | 10回以上の勤務 + 平均評価 3.5以上 |
| Gold | 30回以上 + 平均評価 4.0以上 |
| Platinum | 50回以上 + 平均評価 4.5以上 |

- pg_cron で毎日自動昇格（03:00 JST）
- バッジ（手動付与）も併用可能

### 3-7. 通知システム

| チャネル | 用途 | 実装状況 |
|---------|------|---------|
| メール（Resend） | シフト確定・前日リマインド・出勤確認 | 実装済み |
| LINE Bot | オファー通知・承諾/辞退・リマインド | 実装済み |
| Web アプリ内 | 全通知（notification_logs テーブル） | 実装済み |

### 3-8. 前日リマインド（出勤確認）

```
毎朝 07:00 JST（Cron: /api/cron/reminders）
  → 翌日シフトのトレーナーにメール送信（pre_day_reminder）
  → LINE 連携済みトレーナーには LINE Push も送信（Flex Message）
  → メール: 「OK — 出勤します」ボタン → pre_day_reminder_sent = true
  → LINE: 「確認しました」postback → pre_day_confirmed = true + 打刻URL案内
  → キャンセルボタンなし（Webアプリへ誘導して摩擦を作る設計: 案A）
  → 未確認 → HR ダッシュボードでアラート表示
```

- 毎朝 07:00 JST の Cron ジョブで自動送信（メールと同時）

### 4-5. アカウント連携フロー

```
① トレーナー: Web プロフィール → 「LINE連携」ボタン
   → ワンタイムトークン生成（30分有効）

② トレーナー: LINE で Dr.stretch SPOT を友だち追加
   → Bot が連携案内メッセージを送信

③ トレーナー: LINE で「link:トークン」と送信
   → トークン検証 → alumni_trainers.line_user_id に保存
   → 「LINE連携完了」メッセージ返信

④ 以降、オファー通知が LINE に自動プッシュ
```

### 4-6. Webhook イベント処理

| イベント | 処理内容 |
|---------|---------|
| follow（友だち追加） | 連携済み → おかえりメッセージ / 未連携 → 連携案内 |
| postback: accept_offer | オファー承諾 → シフト確定処理 → 確認メッセージ返信 |
| postback: decline_offer | オファー辞退 → 辞退記録 → 辞退メッセージ返信 |
| postback: confirm_reminder | 前日リマインド確認 → `pre_day_confirmed = true` → 確認メッセージ + 打刻URL返信 |
| message（テキスト） | `link:xxx` → アカウント連携 / その他 → デフォルト応答 |

### 4-7. オファー承諾時の自動処理

LINE Bot から「承諾する」を押した場合:
1. `shift_offers` のステータスを `accepted` に更新
2. `shift_requests` を自動作成（source: `hr_offer` or `direct_offer`）
3. `shift_applications` を自動作成（status: `approved`）
4. `attendance_records` を自動作成（status: `scheduled`）
5. 結果メッセージを LINE で返信

LINE アプリで以下のいずれかの方法で友だち追加:
